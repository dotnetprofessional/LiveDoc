## Session Summary

Investigated and fixed Bug 2: "close timed out after 10000ms" hanging after tests complete with LiveDoc reporter publishing to a server.

## Root Cause Analysis

Node.js's global `fetch()` (undici) uses HTTP keep-alive by default. TCP connections from the reporter's HTTP calls linger in the connection pool, preventing the event loop from draining after tests complete. Vitest waits ~10 seconds then logs "close timed out after 10000ms" before force-exiting.

## Solution Implemented

Added `'Connection': 'close'` header to all `fetch()` calls in:
- `LiveDocViewerReporterV1.post()` — the single HTTP method used by the publish reporter
- `discoverServer()` in `@swedevtools/livedoc-server` — the health check during auto-discovery

## Rationale

- Reporter makes short-lived, one-shot HTTP requests. No benefit to keep-alive.
- Minimal, targeted fix — avoids masking other potential issues with `forceExit: true` or reduced `teardownTimeout`.
- Playwright cleanup (`afterAll → browser.close()`) was already correct and not the cause.

## Impact

- Reporter publish and auto-discovery no longer keep the process alive after tests complete.
- No behavior change for non-publish test runs (no `fetch()` calls made when no server configured).
